Creation of a Capstone

Throughout my time in the Computer Science program, I’ve grown both technically and professionally in ways that go beyond just writing code. Starting the program in 2019 while working full-time in IT, I had to learn how to balance real-world responsibilities with academic expectations. That experience alone helped reshape my work ethics, time management, and ability to stay consistent even when things got challenging. Building my ePortfolio and completing my capstone really brought everything together. I can showcase not just what I have learned, but how I apply those skills in practical, real-world scenarios.

One of the biggest takeaways from this program is how my technical skills have evolved into something more meaningful, something that has provided me with the skills necessary to obtain promotions in my current role. I didn’t just learn how to code during these years at Southern New Hampshire, I learned how to design systems and think critically about problems that actually work in my career. For example, in my full-stack Travlr application, I worked with technologies like Node.js, Express, MongoDB, and Angular to create a complete application from frontend to backend. This included designing a database schema for trips, implementing API endpoints for CRUD operations, and integrating authentication using JWT tokens and password hashing for security. These experiences helped me understand how different parts of a system connect and depend on each other, like a symbiotic relationship.

Collaboration has also been a key part of my growth here. Even though many assignments were completed individually, I developed collaboration skills through peer discussions and applying feedback from instructors each term. In my current IT role, I regularly work with different departments like healthcare staff and law enforcement, where communication is critical. Whether I’m helping a doctor regain access to patient records or troubleshooting issues with police systems, I’ve learned how to communicate technical issues clearly to non-technical stakeholders. That ability to translate complex problems into simple explanations is something I’ve strengthened throughout this program.

Another area where I’ve grown is in my understanding of data structures and algorithms. While working on backend functionalities, I had to think about how data is stored, retrieved, and optimized, which is something we constantly take into account at work. Even something like fetching trip data from an API and handling responses properly required me to think about efficiency and scalability. I’ve learned that writing code isn’t just about making it work.

Software engineering and database design have been major strengths I have also begun to develop. I’ve worked with MongoDB and Mongoose to design schemas, manage connections, and seed databases with data and now I get to implement them on my day to day when working on SQL databases. I’ve also learned how to structure applications. This made my applications more organized and easier to scale. Through debugging and enhancing my projects, I gained a deeper understanding of how to improve existing systems instead of just building new ones from scratch because it’s a real-world skill that is needed, especially when you start a new role in a company that already has an infrastructure built.

Security is another area that stood out during my capstone and that I improved in. Implementing authentication using hashed passwords and JSON Web Tokens gave me hands-on experience with protecting user data and securing APIs. For example, using middleware to verify tokens before allowing access to certain routes helped reinforce the importance of controlling access and preventing unauthorized actions. This aligns closely with my work in IT, where system security and access control are a critical part of my daily responsibilities.

The artifacts in my ePortfolio in many ways reflect these skills. Each one represents a different area of computer science, as well as my personal growth in my career. My software engineering artifact focuses on building and improving a full-stack application. My algorithms and data structures artifact show how I approach efficiency and logic within my code. My database artifact demonstrates how I design and manage data in a structured and meaningful way. While each artifact stands on its own, they all connect through one main built.

Overall, this program has helped me move from simply understanding technology to being able to apply it in a professional setting, something that my current employer has already acknowledged. It has strengthened my ability to solve problems, communicate effectively, and think like a modern developer. As I move forward in my career, I plan to continue building on these and more skills and hopefully creating technology that is both functional and accessible to the people who use it.
